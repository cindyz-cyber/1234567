## 音频文件规格

### 推荐规格

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 格式 | MP3 | 兼容性最好 |
| 比特率 | 192kbps | 高品质，文件大小适中 |
| 采样率 | 44.1kHz | 标准音质 |
| 时长 | 10-30 分钟 | 循环播放，覆盖整个流程 |
| 文件大小 | < 100MB | 系统上传限制 |

### 文件大小参考

| 时长 | 192kbps | 320kbps |
|------|---------|---------|
| 10 分钟 | ~14MB | ~24MB |
| 20 分钟 | ~29MB | ~48MB |
| 30 分钟 | ~43MB | ~72MB |
| 60 分钟 | ~86MB | ~144MB ❌ 超限 |

## 性能优化

### 自动流式播放

系统自动为所有音频启用流式播放优化：

```
传统方式:
  下载完整文件 (100MB)
  ↓ 等待 30-60 秒
  开始播放

流式优化:
  预加载元数据 (1-2MB)
  ↓ 等待 1-2 秒
  开始播放 + 边下载边播放
```

**性能提升**:
- 🚀 首次播放时间: 30-60 秒 → 1-2 秒
- 💾 内存占用: 100MB → 10-20MB
- 📡 首屏流量: 100MB → 1-2MB

### 微信浏览器兼容

系统使用 Supabase Storage，原生支持：
- ✅ HTTP 206 Range Requests
- ✅ CORS 跨域访问
- ✅ 微信安全域验证

## 故障排查

### 问题 1: 音频没有播放

**可能原因**:
1. 场景的 `bg_music_url` 为空 + 主 App 没有可用音频

**解决方案**:
- 访问 `/admin/audio-uploader` 上传至少一个音频文件
- 或为场景配置专属音频

---

### 问题 2: 音频播放卡顿

**可能原因**:
1. 音频文件过大（> 100MB）
2. 网络速度慢

**解决方案**:
- 使用 192kbps 而非 320kbps（减小文件 40%）
- 将音频时长控制在 30 分钟以内
- 使用 CDN 加速

---

### 问题 3: 新场景显示"访问被拒绝"

**可能原因**:
1. Token 验证失败
2. 场景已停用

**解决方案**:
- 检查 URL 中的 `token` 参数是否正确
- 在管理后台确认场景已启用

---

### 问题 4: 上传音频失败

**可能原因**:
1. 文件大于 100MB
2. 文件不是 MP3 格式

**解决方案**:
- 使用 192kbps 重新导出音频
- 确认文件扩展名为 `.mp3`

## 数据库直接操作（高级）

### 查询所有场景

```sql
SELECT scene_token, scene_name, bg_music_url, is_active
FROM h5_share_config
ORDER BY created_at;
```

### 更新场景音频

```sql
UPDATE h5_share_config
SET bg_music_url = 'https://your-cdn.com/new-music.mp3'
WHERE scene_token = 'morning';
```

### 停用场景

```sql
UPDATE h5_share_config
SET is_active = false
WHERE scene_token = 'morning';
```

## 最佳实践

### 1. 场景命名规范

✅ **推荐**:
- `morning` - 晨间唤醒
- `healing` - 深度疗愈
- `sleep` - 睡前放松
- `meditation` - 冥想练习

❌ **不推荐**:
- `scene1`, `scene2` - 无语义
- `测试场景` - 包含中文
- `new-scene-2024-03-09` - 过于冗长

### 2. Token 管理

每个场景使用不同的 `daily_token`，便于：
- 渠道追踪
- 访问控制
- 定期更换（防止链接泄露）

### 3. 音频内容匹配

确保音频内容与场景主题匹配：

| 场景 | 音频类型 | 特点 |
|------|----------|------|
| 晨间唤醒 | 轻快、振奋 | 高频能量音乐 |
| 深度疗愈 | 舒缓、平静 | 低频疗愈音乐 |
| 睡前放松 | 柔和、催眠 | 白噪音、自然声 |
| 冥想练习 | 空灵、专注 | 钵音、禅修音乐 |

### 4. 定期维护

- 每月检查主 App 音频库（确保至少 1 个可用音频）
- 每季度更新场景音频（保持新鲜感）
- 监控访问日志（了解各场景使用情况）
