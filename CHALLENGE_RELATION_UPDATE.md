# 反作用力/挑战关系检测系统

## 功能概述

新增了对马雅历中「挑战关系」（反作用力场域）的自动检测和深度解读功能。当家族成员之间的Kin差值恰好为130时，系统会识别这种特殊的极性关系，并生成专门的能量分析报告。

## 核心逻辑

### 1. 检测规则

```typescript
// 在 detectRelationshipSynergy 函数中
const kinDiff = Math.abs(userKin.kin - relativeKin.kin);

if (kinDiff === 130) {
  return {
    hasSynergy: true,
    type: 'challenge',
    strength: 1.0,
    isChallengeRelation: true,
    tensionLevel: 0.95
  };
}
```

**判定条件：** `Math.abs(kinA - kinB) === 130`

**示例：**
- Kin 65 和 Kin 195 → 差值130 → 挑战关系 ✓
- Kin 1 和 Kin 131 → 差值130 → 挑战关系 ✓
- Kin 200 和 Kin 70 → 差值130 → 挑战关系 ✓

### 2. 能量修正逻辑

在 `calculateFamilyCollision` 函数中：

```typescript
if (isChallengeRelation) {
  // 挑战关系特殊处理
  adjusted.pineal += 8;   // 增强直觉与自我意识
  adjusted.throat += 5;   // 增强独立表达能力
  // 心轮不增加，保持原始状态（带张力）
} else {
  // 正常家族关系的能量加成逻辑
  // ...
}
```

**设计理念：**
- **心轮**：保持初始分，不增加也不减少（体现张力状态）
- **松果体**：+8% 增强自我意识与直觉洞察
- **喉轮**：+5% 增强独立表达与自我声音

### 3. 类型系统扩展

```typescript

  hasSynergy: boolean;
  type: 'mutual-push' | 'harmonic-resonance' | 'energy-amplify' | 'challenge' | null;
```

## 技术实现细节

### 1. 检测优先级

在 `detectRelationshipSynergy` 中，挑战关系检测优先于其他共振类型：

```typescript
// 首先检测「挑战关系」
if (kinDiff === 130) {
  return { type: 'challenge', ... };
}

// 然后检测其他类型
if (sealDiff <= 2 && toneDiff <= 1) {
  return { type: 'mutual-push', ... };
}
// ...
```

### 2. 多人叠加处理

如果用户添加了多个家族成员：
- 系统会独立检测每对关系
- 如果其中一个是挑战关系，会在报告中突出显示
- 其他正常关系仍会正常显示

### 3. 张力等级

```typescript
tensionLevel: 0.95  // 挑战关系的张力设为95%（极高）
```

这个值可用于未来的可视化展示（如雷达图中的特殊标识）。

## 边界情况处理

### 1. 跨越260边界

```typescript
// Kin 10 和 Kin 140
差值 = |10 - 140| = 130 ✓ 触发挑战关系

// Kin 250 和 Kin 120
差值 = |250 - 120| = 130 ✓ 触发挑战关系
```

### 2. 精确匹配

只有**恰好130**才触发，129或131都不会：

```typescript
kinDiff === 130  // 必须精确相等
```

### 3. 双印记（子时出生）

如果用户选择了前子时或后子时：
- 系统会基于主Kin计算挑战关系
- secondaryKin暂不参与挑战关系检测
