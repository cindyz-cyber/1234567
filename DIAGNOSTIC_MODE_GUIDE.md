# 声音分析诊断模式使用指南

## 概述

诊断模式已内置到声音分析系统中，每次进行声音分析时会自动在浏览器控制台输出详细的排查报告。

## 如何查看诊断报告

1. 打开浏览器开发者工具（F12 或右键 > 检查）
2. 切换到 Console（控制台）标签页
3. 进行声音录制和分析
4. 查看控制台输出的详细诊断信息

## 诊断报告内容

### 1. 原始频谱分布 (Raw Spectrum)

显示声音在各频段的能量分布百分比：

```
📊 原始频谱分布 (Raw Spectrum):
   100-200Hz: XX.XX%
   200-300Hz: XX.XX%
   300-400Hz: XX.XX%
   400-500Hz: XX.XX%
   500-600Hz: XX.XX%
   600Hz+: XX.XX%
```

**解读要点**：
- 100-300Hz 占比高 → 低频主导，稳健型
- 400-500Hz 占比高 → 中高频主导，可能为心轮/喉轮
- 600Hz+ 占比高 → 高频主导，灵性/头脑型

### 2. 基频偏移检查

```
🎯 基频偏移检查:
   检测到的主导频率: XXXHz
   260Hz附近能量: XX.XX%
   432Hz附近能量: XX.XX%
   是否为低频主导: ✓ 是 / ✗ 否
```

**解读要点**：
- 主导频率在 200-300Hz → 应该触发稳健型标签
- 主导频率在 400Hz+ → 可能触发心轮/喉轮型
- 260Hz 能量高但未识别 → 可能存在算法偏差

### 3. 权重调整策略

系统已应用以下权重修正（针对"紫色静默连接者"误判问题）：

```
⚖️ 权重调整策略:
   - 紫色/灵性维度权重: -30%
   - 下三轮（100-300Hz）权重: +30%
```

**说明**：
- Crown（顶轮）和 ThirdEye（眉心轮）能量 × 0.7
- Root（根轮）和 Sacral（脐轮）能量 × 1.3
- 低频主导时，Root 额外提升 20%

### 4. 调整后脉轮能量

```
🔋 调整后脉轮能量:
   root: XX.XX
   sacral: XX.XX
   solar: XX.XX
   heart: XX.XX
   throat: XX.XX
   thirdEye: XX.XX
   crown: XX.XX
```

**关键指标**：
- Root > 30 → 具备触发"稳健共振师"的基础条件
- Crown/ThirdEye 过高 → 可能误判为灵性型

### 5. 标签触发权重诊断

```
🏷️ 标签触发权重诊断:
   匹配到的标签: XXX
   原型ID: XXX
   匹配度: XX.X%

   与【稳健共振师】对比:
   【稳健共振师】触发阈值:
     - Root脉轮能量 > 30
     - 主导频率 200-300Hz
     - 相位: grounded
     - 质地: smooth/balanced

   实际值对比:
     - Root能量: XX.XX ✓/✗
     - 频率匹配: XXXHz ✓/✗
     - 相位匹配: XXX ✓/✗
     - 质地匹配: XXX ✓/✗
```

**解读要点**：
- 4 个条件都是 ✓ → 应该匹配"稳健共振师"
- 某项是 ✗ → 说明该项不满足阈值，需要进一步调整

### 6. 强制修正机制

当检测到明显的低频主导但未匹配到稳健型标签时，系统会自动应用强制修正：

```
🔄 应用【扎根与稳健】强制修正:
   原标签: 紫色静默连接者
   修正后: 稳健共振师
   修正原因: 低频主导 + Root能量充足
```

**触发条件**：
1. 低频（100-300Hz）占比 > 40%
2. Root 脉轮能量 > 25
3. 主导脉轮为 Root 或 Sacral

## 声学特征分析（多维审计）

除了频谱诊断，系统还会输出声学特征：

```
[VoiceAnalyzer] Running acoustic feature extraction
[VoiceAnalyzer] Roughness: 45.2 Warning: false
```

- **Roughness < 60%**：声音通透，无应激
- **Roughness 60-80%**：中度刺耳，存在紧张
- **Roughness > 80%**：严重刺耳，强烈防御

## 常见问题排查

### Q1: 为什么 260Hz 的声音被识别为"紫色静默连接者"？

**排查步骤**：
1. 检查"原始频谱分布"，确认 200-300Hz 是否为主导频段
2. 检查"调整后脉轮能量"，确认 Root 是否 > 30
3. 检查是否触发了"强制修正"

**可能原因**：
- 其他频段干扰过强（如 400Hz+ 能量也很高）
- Root 权重提升不足
- 音质问题导致频谱分散

### Q2: 如何确认权重调整是否生效？

对比"调整前"和"调整后"的脉轮能量：
- ThirdEye 和 Crown 应该比原始值低 30%
- Root 和 Sacral 应该比原始值高 30%
- 如果低频主导，Root 还应额外提升 20%

### Q3: 强制修正何时触发？

必须同时满足：
1. 低频占比（100-300Hz）> 40%
2. Root 能量 > 25
3. 主导脉轮是 Root 或 Sacral

如果不满足条件，即使调整权重也不会触发强制修正。

## 调试建议

1. **录制多个样本**：同一个人录制 3-5 次，观察频谱是否一致
2. **对比不同人**：Cindy vs 其他人，确认算法对不同音色的响应
3. **调整环境**：降低背景噪音，使用更好的麦克风
4. **手动验证**：使用音频分析软件（如 Audacity）查看真实频谱

## 技术实现位置

所有诊断代码在以下文件中：

- **主文件**: `src/utils/voiceAnalysis.ts`
  - `analyzeSpectrumDistribution()` - 频谱分布分析
  - `extractChakraEnergyWithDiagnostics()` - 带权重调整的能量提取
  - 强制修正逻辑在 `analyzeAudioBuffer()` 第 8 步

- **特征提取**: `src/utils/acousticFeatureExtractor.ts`
  - 多维审计协议
  - 粗糙度分析
  - 健康预警生成
