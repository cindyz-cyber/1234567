# ShareJournal 页面隐藏功能使用指南

## 功能概述

管理员现在可以在两个维度控制 ShareJournal 引流页面的可见性：

1. **整个场景的显示/隐藏**（场景级别） - 在多场景配置后台操作
2. **场景内单个子页面的显示/隐藏**（页面级别） - 在页面文案配置后台操作

---

## 两种隐藏功能的区别

| 功能 | 操作入口 | 控制粒度 | 用途 |
|------|---------|---------|------|
| **场景隐藏** | `/admin/share-config` | 整个场景 | 临时关闭整个引流页面活动 |
| **子页面隐藏** | `/admin/share-config`（页面文案配置） | 单个子页面 | 灵活定制用户流程，跳过某些步骤 |

---

## 功能一：场景级别隐藏（整个引流页面）

### 操作入口

访问多场景配置后台：`/admin/share-config`

### 如何使用

#### 方式一：场景列表快速切换（推荐）

1. **访问多场景配置后台**
   ```
   https://你的域名/admin/share-config
   ```

2. **输入管理员密码**
   ```
   plantlogic2026
   ```

3. **在场景列表中快速切换**

   每个场景卡片右侧有：
   - 👁️ **眼睛图标按钮**：点击切换显示/隐藏
   - 🟢/🔴 **状态圆点**：绿色=显示中，红色=已隐藏

   **操作步骤**：
   - 点击绿色眼睛图标 → 页面立即隐藏（变成红色关闭眼睛）
   - 点击红色关闭眼睛图标 → 页面立即显示（变成绿色眼睛）

   **特点**：
   - ✅ 无需进入编辑表单
   - ✅ 即时生效，自动刷新
   - ✅ 保持当前选中的场景

---

#### 方式二：编辑表单中的状态按钮

1. **选择要操作的场景**

   在左侧场景列表中点击场景卡片

2. **使用顶部状态按钮**

   在编辑表单顶部有一个大按钮：
   - 🟢 **「页面显示中」**（绿色）→ 点击隐藏页面
   - 🔴 **「页面已隐藏」**（红色）→ 点击显示页面

3. **按钮会自动更新**

   点击后状态立即切换，无需手动保存

---

#### 方式三：编辑表单中的复选框

1. **选择场景并向下滚动到「启用场景」区域**

2. **切换「启用此场景」复选框**
   - ☑️ 勾选 = 页面显示
   - ☐ 取消勾选 = 页面隐藏

3. **点击「保存配置」按钮**

   使用此方式需要手动保存才能生效

---

### 用户访问被隐藏场景时看到什么

当用户访问被隐藏的页面时，会看到：

```
🔒

此页面已暂时关闭

场景「测试场景」已被管理员隐藏
请稍后再试或联系管理员

✨
植本逻辑
觉察 · 疗愈 · 成长
```

## 功能二：子页面级别隐藏（流程步骤定制）

### 操作入口

1. 访问多场景配置后台：`/admin/share-config`
2. 点击右上角「页面文案配置」按钮
3. 或直接访问：`/admin/page-content`

### 可隐藏的子页面

引流流程包含以下页面，每个都可以独立隐藏：

| 页面英文名 | 页面中文名 | 说明 |
|-----------|----------|------|
| naming | 起名页 | 输入姓名和生日 |
| home | 首页 | 欢迎页面 |
| emotion | 情绪扫描 | 选择当前情绪 |
| journal | 日记页 | 记录内心想法 |
| dialogue | 高我对话 | 与高我对话环节 |
| transition | 过渡页 | 黄金转场动画 |
| answer | 答案之书 | 显示高我指引 |
| card | 能量卡片 | 最终生成分享卡片 |

### 如何隐藏子页面

1. **进入页面文案配置后台**
   ```
   /admin/page-content
   ```

2. **输入场景标识**

   在顶部输入要配置的场景标识（如 `default`、`1234test` 等）

3. **找到要隐藏的页面组**

   页面按组显示，每组对应一个子页面

4. **点击页面组右侧的按钮**

   每个页面组标题旁有显示/隐藏按钮：
   - 👁️ **「显示中」**（绿色）→ 点击隐藏该页面
   - 👁️‍🗨️ **「已隐藏」**（红色）→ 点击显示该页面

5. **立即生效**

   无需保存，点击后立即生效

### 隐藏子页面的效果

被隐藏的子页面会在用户流程中**自动跳过**：

**示例 1**：隐藏「首页」
```
用户流程：起名页 → 情绪扫描 → 日记页 → ...
          （跳过首页）
```

**示例 2**：隐藏「情绪扫描」和「过渡页」
```
用户流程：起名页 → 首页 → 日记页 → 高我对话 → 答案之书 → 卡片
          （跳过情绪扫描和过渡页）
```

**示例 3**：只保留核心流程
```
隐藏：首页、情绪扫描、过渡页
保留：起名页 → 日记页 → 高我对话 → 答案之书 → 卡片
```

### 注意事项

1. **至少保留一个页面可见**

   不能隐藏所有页面，建议至少保留核心流程

2. **卡片页通常不隐藏**

   卡片页是最终结果，通常应该保持可见

3. **流程会自动调整**

   系统会自动跳过被隐藏的页面，无需手动调整代码

4. **每个场景独立配置**

   不同场景可以有不同的页面可见性配置

## 使用场景

### 场景级别隐藏的使用场景

1. **活动结束**
   - 限时活动结束后，临时关闭整个引流页面
   - 保留配置，方便下次复用

2. **内容更新**
   - 需要修改配置或内容时
   - 避免用户访问到未完成的内容

3. **AB 测试**
   - 快速切换不同版本进行测试
   - 无需删除配置即可暂停某个版本

4. **流量控制**
   - 控制不同渠道的访问
   - 灵活调整推广策略

### 子页面级别隐藏的使用场景

1. **简化流程**
   - 对于某些渠道，跳过不必要的步骤
   - 例如：直接从起名页跳到日记页

2. **定制化体验**
   - 为不同用户群体提供不同的流程
   - 例如：新用户显示首页，老用户跳过首页

3. **功能测试**
   - 临时关闭某个有问题的步骤
   - 避免影响整体流程

4. **渐进式发布**
   - 新功能先隐藏，测试通过后再显示
   - 逐步开放新的页面功能

5. **快速优化**
   - 根据数据分析，隐藏转化率低的页面
   - 优化用户路径

## 数据库字段

### 场景级别（h5_share_config 表）

- `is_active = true` → 整个场景显示
- `is_active = false` → 整个场景隐藏

### 页面级别（page_visibility_config 表）

- `is_visible = true` → 该页面在流程中显示
- `is_visible = false` → 该页面被跳过

可以直接在数据库中修改这些字段，或通过后台界面操作。

---

## 注意事项

1. **隐藏不是删除**
   - 隐藏的场景配置仍然存在
   - 所有媒体资源、文案配置都保留
   - 随时可以重新显示

2. **立即生效**
   - 快速切换按钮操作后立即生效
   - 用户刷新页面即可看到新状态

3. **不影响其他场景**
   - 每个场景独立控制
   - 隐藏一个不影响其他场景

4. **默认场景建议**
   - 建议不要隐藏 `default` 场景
   - 保持至少一个可访问的场景

## 快速操作流程

### 快速隐藏整个场景

1. 访问 `/admin/share-config`
2. 输入密码 `plantlogic2026`
3. 在场景列表找到要隐藏的场景

1. 访问 `/admin/page-content`
2. 输入密码 `plantlogic2026`
